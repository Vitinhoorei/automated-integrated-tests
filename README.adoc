== 🇧🇷 Versão em Português

=== 📝 Descrição

O **SAP AI-Driven Automation Tester** é uma solução desenvolvida em **Python** para automatizar testes integrados no sistema **SAP**, com foco em processos do módulo **PM (Plant Maintenance)**.

O sistema combina:

* **Automação de Interface SAP GUI (SAP GUI Scripting)**
* **Processamento com Inteligência Artificial**
* **Execução automatizada de cenários de teste**
* **Captura estruturada de evidências**

A ferramenta é capaz de:

* interpretar cenários definidos em **planilhas Excel**
* executar **transações SAP automaticamente**
* capturar **prints e evidências**
* identificar **erros técnicos ou dumps**
* gerar **relatórios estruturados**

---

=== 🎯 Problema e Impacto

Tradicionalmente, a validação de fluxos de manutenção no SAP exige grande esforço manual.

**Cenário anterior**

* Execução manual de testes
* Captura manual de evidências
* Alto risco de erro humano

O ciclo completo de testes podia levar cerca de:

**6 semanas**

**Com a automação**

O sistema executa os testes automaticamente e gera as evidências.

O tempo foi reduzido para aproximadamente:

**até 2 dias**

**Resultado**

Redução superior a **95% no tempo operacional**, além de:

* padronização das evidências
* redução de erros humanos
* maior rastreabilidade dos testes

---

=== ⚙️ Funcionalidades Principais

* **Integração SAP GUI**  
Automação nativa utilizando **SAP GUI Scripting (win32com)**.

* **Execução Automática de Transações**  
Execução automática de TCODEs com preenchimento de campos.

* **Captura de Evidências**  
Geração automática de prints de tela e dumps estruturados.

* **Memória de Contexto**  
Reutilização de IDs gerados (ex: Notas ou Ordens) em etapas posteriores.

* **Relatório Inteligente**  
Geração de relatórios em **Excel** com logs detalhados.

* **Análise de Erros**  
Em caso de erro ou dump SAP, o sistema coleta evidências e envia para análise.

=== ⚙️ Como Executar o Projeto

==== 1️⃣ Clonar o repositório

[source,bash]
----
git clone https://github.com/Vitinhoorei/automated-integrated-tests.git
cd automated-integrated-tests
----

==== 2️⃣ Criar ambiente virtual

[source,bash]
----
python -m venv venv
----

Ativar o ambiente:

Windows

[source,bash]
----
venv\Scripts\activate
----

Linux / Mac

[source,bash]
----
source venv/bin/activate
----

==== 3️⃣ Instalar dependências

As dependências estão definidas no arquivo **requirements.txt**

[source,bash]
----
pip install -r requirements.txt
----

==== 4️⃣ Executar o sistema

[source,powershell]
----
python src/main.py --file "c:\caminho\para\arquivo.xlsx" --sheet "NomeDaAba"
----

Exemplo:

[source,powershell]
----
python src/main.py --file "C:\Users\veleoterio\Downloads\Cenarios_PM.xlsx" --sheet "Testes unitarios"
----

== 🇺🇸 English Version

=== 📝 Description

The **SAP AI-Driven Automation Tester** is a **Python-based automation framework** designed to execute integrated tests within the **SAP system**, focusing on the **Plant Maintenance (PM)** module.

The solution combines:

* **SAP GUI interface automation**
* **Artificial Intelligence**
* **Automated execution of test scenarios**
* **Structured evidence collection**

The system can:

* interpret **Excel-based test scenarios**
* execute **SAP transactions automatically**
* capture **screenshots and technical dumps**
* analyze **errors and system failures**
* generate **structured Excel reports**

---

=== 🎯 Problem and Impact

Traditionally, validating SAP maintenance flows required a significant amount of manual work.

**Previous scenario**

* Manual execution of transactions
* Manual evidence collection
* High risk of human errors

A complete testing cycle could take around:

**6 weeks**

**With automation**

The automated system executes scenarios and captures all required evidence.

Execution time was reduced to approximately:

**2 days**

**Result**

More than **95% reduction in operational time**, along with:

* standardized evidence
* reduced human errors
* improved test traceability

---

=== ⚙️ Key Features

* **SAP GUI Integration**  
Native automation using **SAP GUI Scripting (win32com)**.

* **Automatic Transaction Execution**

* **Evidence Capture**
Automated screenshots and structured screen dumps.

* **Context Memory**
Reuses generated IDs (e.g., Notifications or Orders) between steps.

* **Smart Reporting**
Excel output with detailed logs.

* **Error Analysis**
Technical failures and dumps can be analyzed automatically.
